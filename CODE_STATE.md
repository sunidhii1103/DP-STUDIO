# CODE_STATE

## 1. Project Summary
A frontend-only React application for visualizing Dynamic Programming algorithms using a strict step-driven architecture. The UI purely renders discrete states (Steps) generated by an independent algorithm Step Engine, ensuring no algorithm logic resides in the React components.

## 2. Architectural Constraints (Step Engine Rules)
- **4-Layer Architecture**:
  - Layer 1 (Algorithm Logic): Pure functions generating arrays of `Step` objects.
  - Layer 2 (Step Engine): Orchestrates step generation & manages comparison milestones.
  - Layer 3 (State Management): Centralized `AppContext` + `AppReducer`.
  - Layer 4 (UI Rendering): Strictly reads from state. Upward imports are forbidden.
- **Step-Driven Data Flow**: UI reads a single unified `Step` object (`TableSnapshot`, `ActiveIndices`, `Explanation`, `CodeReference`). Zero inter-panel communication.
- **Animations**: Driven exclusively by CSS class changes based on the active step's state. No JS-based direct DOM animations.
- **Performance**: Strict use of `React.memo` on UI panels to prevent unnecessary re-renders with large algorithm inputs.

## 3. Planned File Structure
```text
src/
├── algorithms/      # Layer 1 (Fibonacci, Knapsack, LCS, registry)
├── engine/          # Layer 2 (StepOrchestrator, ComparisonOrchestrator)
├── state/           # Layer 3 (AppContext, AppReducer, actions, selectors)
├── hooks/           # Layer 3/4 bridge (useCurrentStep, usePlayback, etc.)
├── components/      # Layer 4 (UI)
│   ├── viz/         # DPCell, DPTable1D, DPTable2D
│   ├── controls/    # PlaybackControls, InputPanel
│   └── panels/      # ExplanationPanel, CodePanel, ComparisonView, ComplexityChart
├── pages/           # Pages & Routing (Home, Visualization, Learning, Debug)
├── types/           # TS Interfaces (Step schema, UI types)
└── styles/          # Vanilla CSS system (premium design & transitions)
```

## 4. Core Modules & Responsibilities
1. **Step Generation Engine (L1/L2)**: Generates pure `Step[]` arrays from inputs.
2. **Visualization Module (L4 `viz/`)**: Renders DP tables and highlights cells based on active step data.
3. **Control System (L4 `controls/`)**: Manages playback (play/pause/step/speed).
4. **Explanation Engine (L4 `panels/ExplanationPanel`)**: Displays step reasoning and metadata.
5. **Code Synchronization Module (L4 `panels/CodePanel`)**: Displays and highlights algorithm code based on `CodeReference`.
6. **Comparison Engine (L2/L4)**: Orchestrates and renders side-by-side execution of different approaches.
7. **Complexity Visualization Module (L4 `panels/ComplexityChart`)**: Renders execution time vs input size using native SVG/Canvas.
8. **Learning Mode (L4 `pages/LearningPage`)**: Dedicated view explaining DP concepts (overlapping subproblems, optimal substructure).
9. **Step Inspection & Debug Mode (L4 `pages/DebugPage`)**: Advanced view to inspect precise state snapshots and jump between steps.

## 5. Dependencies
- **react, react-dom**: Core rendering.
- **react-router-dom**: Client-side routing.
- **react-resizable-panels**: Resizable UI layouts.
*(No external charting, styling, or state-management libraries needed. Vanilla CSS, Canvas/SVG, Context+useReducer will be used).*
